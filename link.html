  6. What is a file path? How does it differ from a URL?

    Answer..A file path is a specific location within a file system that points to a file or directory. It specifies the precise route to reach a file or directory on a computer or server. File paths are typically written using a hierarchical structure, such as directories separated by slashes ("/") or backslashes (""). A URL (Uniform Resource Locator), on the other hand, is a specific type of file path that includes additional information needed to access resources over the internet. It consists of several components, including the protocol (e.g., HTTP, HTTPS), domain name, and optionally a path to a specific resource on the server. URLs often begin with "http://" or "https://" and are used to uniquely identify resources on the web. While a file path refers to a location within a file system, a URL points to a resource accessible over the internet.

  7. Explain the significance of the root directory in a file

    Answer..The root directory is the top-most directory in a file system hierarchy. It serves as the starting point for navigating the entire file system. In Unix-like systems, it's represented by a single forward slash ("/"), while in Windows, it's usually denoted by the drive letter followed by a colon (e.g., "C:"). Understanding the root directory is crucial because all other directories and files are organized hierarchically beneath it. It provides a frame of reference for specifying the location of any file or directory within the file system.

  14. Describe the purpose of the action attribute in a form tag.

    Answer..The action attribute in a form tag specifies the URL where the form data will be sent upon submission. It defines the endpoint or destination where the form data will be processed, usually by a server-side script or application.

  15. What is the significance of the method attribute in a form tag?

    Answer..The method attribute in a
    tag specifies the HTTP method used when submitting the form data to the server. It defines how the form data will be sent. The two most commonly used methods are GET and POST. GET method appends the form data to the URL in the form of query parameters, visible in the address bar. It is typically used for requests that retrieve data from the server. POST method sends the form data in the request body, which is not visible in the URL. It is used for requests that submit data to the server, often for actions that modify data on the server. Other less commonly used methods include PUT, DELETE, and PATCH, which are often used in RESTful APIs for CRUD operations.

  17. Explain the difference between the GET and POST methods in form submission

    Answer..The main difference between the GET and POST methods in form submission lies in how the data is sent to the server: GET Method: Data is appended to the URL as a query string. Limited amount of data can be sent (typically up to 2048 characters). Data is visible in the URL, which can pose security risks, especially for sensitive information like passwords. Suitable for sending small amounts of non-sensitive data, such as search queries. Data is cached by the browser, which may lead to security concerns if the data contains sensitive information. POST Method: Data is sent in the HTTP request body. Can send larger amounts of data compared to GET method. Data is not visible in the URL, enhancing security and privacy. Suitable for sending sensitive information like passwords or personal details. Data is not cached by the browser, reducing security risks associated with storing sensitive information.
